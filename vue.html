路由

  • 作用:

    • 前端路由:

      单页面应用(SPA)，当触发了不同的链接，展现不同的内容(组件)

    • 后台路由

      分门别类的处理浏览器请求

    • Angular中的路由视图

      ngRoute
html : ng-view   
js: 处理路由

    • Vue

      html :
    router-view  
        <router-link>

js：
    1、组件定义好
    2、创建路由对象，设置路由规则
    3、把路由对象注入到根实例，让整个应用拥有路由的功能

  • Vue路由案例(不带参数)

    <script src="../vue.js"></script>
<script src="../vue-router.js"></script>

<body>
<div id="app">
    <router-link to="/goodslist">商品列表</router-link>
    <router-link to="/newslist">新闻列表</router-link>

    <!-- 相当于ng-view -->
    <!-- 路由匹配到的组件将渲染在这里 -->
    <router-view></router-view>
</div>
</body>

<script>
//1.定义好商品组件 & 新闻组件
var goodsListComponent = Vue.extend({
    template:"<ul><li>秋裤</li><li>毛裤</li><li>貂皮内裤</li></ul>"
})

var newsListComponent = Vue.extend({
    template:"<ul><li>十九大18好召开</li><li>以色列跑路了</li><li>金正恩要滑雪了</li></ul>"
})

//2.创建路由对象
var router = new VueRouter({
    //设置路由规则
    routes:[
        {path:"/",redirect:'/goodslist'},
        {path:"/goodslist",component:goodsListComponent},
        {path:"/newslist",component:newsListComponent}
    ]
})

//3.在根实例中注入路由对象，让整个应用都拥有路由功能
var vm = new Vue({
    el:"#app",
    router:router
})  
</script>

父子件集成和通信（传值给子组件）

只有通过components注册的，才能成为父子关系

  1. 写好一个Vue文件，在需要用到的地方引入
    import subcomment from './subcomment.vue'
  2. 注册子组件
    components:{
        subcomment:subcomment
}
  3. 使用子组件并且传递需要的值
    <subcomment :commentId='this.$route.params.newsId'></subcomment>
  4. 子组件接收父组件传递的值
    props: ['commentId']
    props和data、methods同级，属于Vue实例中的一个对象，把父元素中传递过来的值声明到数组中，需要用引号

  5. 使用传递过来的值

    this.commentId

    commentId不需要在data中声明，因为需要props声明

    子组件如何传值给父组件

Vuex

  • 用于非父子组件之间传值，在各自组件之间进行数据的共享，专门用来给Vue进行全局数据管理
  • 什么时候用Vuex https://vuex.vuejs.org/zh-cn/intro.html

  • Vuex核心概念

    • state:数据，里面是键值对
    • getter:获取数据用的，里面是函数
    • mutations:同步的保存或是更新仓库中的数据，里面也是函数
    • action:异步的保存或是更新仓库中的数据，里面也是函数

    注意:

    1. state、getter、mutations、action、module都是对象
    2. state、module对象里面写的是键值对
    3. getter、mutations、action对象里面写的是函数
    4. 一般在做耗时操作的时候，使用actions，如果不是耗时的任务使用mutations即可
    5. getter和mutations函数中的第一个参数必须是state
    6. getter必须要有返回值(return)，调用的时候不要加()
    7. mutations调用的时候用$store.commit
    8. actions调用的时候用的是$store.dispatch

  • 简单使用

    1. 先下载并且引入vuex，在主函数文件中use(Vuex)

      import Vuex from 'vuex';
Vue.use(Vuex)

    2. 在入口函数中实例化Vuex，并且在mutations(同步传递)中声明触发的回调函数

      const store = new Vuex.Store({
state:{
    goodslist:[]
},
getters:{
    getGoodsCount(state){
        let goodsCount=0;
        state.goodslist.forEach(item=>{
            goodsCount+=item.count;
        })
        return goodsCount;
    },
},
mutations: {//同步保存和更新state中的数据
    joinGoods(state,goods) {
        state.goodslist.push(goods);
    }
},
actions:{//异步保存和更新state中的数据
    asyncSaveGoods(context,goods) { //context 相当于$store
        context.commit('joinGoods',goods) //调用同步保存商品的方法
    }
}});

    3. 在需要传递值的组件中使用 v-on的方式绑定事件来传递值

      <button @click='joinCar'>按钮</button>

    4. 使用this.$store.commit把需要传递的值放入Vuex中

      joinCar(){
    this.$store.commit("joinGoods",{count:this.joinNumber,id:this.$route.params.goodId});
}

    5. 在需要的地方通过this.$store.getters.获取

      this.$store.getters.getGoodsList
